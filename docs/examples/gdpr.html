GDPR Banner Component

The GDPR banner is a fixed-position consent notification displayed at the bottom of the page. It includes responsive design, customizable colors via CSS variables, and fade-out animations.

Basic Usage

The banner consists of an outer container (`.gdpr-bottom-box`) and an inner card (`.gdpr-bottom-box-cont`). The `#gdpr-box` id with `.noshow` modifier hides the banner.

<div id="gdpr-box" class="gdpr-bottom-box"> <div class="gdpr-bottom-box-cont"> <p>This site uses cookies to provide services and analyze traffic.</p> <button type="button">Accept</button> </div> </div>
Live demo: Scroll down to see the banner at the bottom of the viewport.

Classes

  • .gdpr-bottom-box — outer fixed container, positioned at bottom with full width
  • .gdpr-bottom-box-cont — inner card; centered, max-width 960px on desktop, 100% on mobile (≤767.98px)
  • #gdpr-box — the banner's id, used for JavaScript interaction
  • #gdpr-box.noshow — modifier class that fades out the banner and disables pointer events

Color Customization (CSS Variables)

The banner supports customization via CSS custom properties. Projects can override any of these to match their brand:

  • --gdpr-bg — banner background (default: #f7f7f7)
  • --gdpr-border — banner border color (default: #e5e5e5)
  • --gdpr-text — text/paragraph color (default: #292929)
  • --gdpr-button-bg — button background (default: #d1d1d1)
  • --gdpr-button-border — button border (default: #cbcbcb)
  • --gdpr-button-text — button text color (default: #212121)
  • --gdpr-button-bg-hover — button background on hover (default: #616161)
  • --gdpr-button-text-hover — button text on hover (default: #fff)
/* Example: Dark theme */ #gdpr-box { --gdpr-bg: #1a1a1a; --gdpr-border: #333; --gdpr-text: #f0f0f0; --gdpr-button-bg: #007bff; --gdpr-button-text: #fff; }

Responsiveness

The banner is fully responsive:

  • Desktop (≥768px): Card has max-width of 960px, centered on page
  • Mobile (≤767.98px): Card takes full width with padding, constrained by viewport
Test responsiveness: Open this page on different screen sizes or use your browser's responsive design mode (F12) to see the banner adapt.

JavaScript Integration

The consuming project's JavaScript is responsible for:

  • Showing/hiding the banner based on consent state
  • Adding the .noshow class to fade out and hide the banner after the user accepts
  • Persisting consent state (e.g., in localStorage)
// Example: Hide banner on button click const gdprBox = document.getElementById('gdpr-box'); const acceptButton = gdprBox.querySelector('button'); acceptButton.addEventListener('click', () => { gdprBox.classList.add('noshow'); localStorage.setItem('gdpr-accepted', 'true'); });

Styling Notes

  • Z-index: Set to 1000, appropriate for fixed overlays but below modals
  • Position: Fixed at the bottom of the viewport, always visible when not hidden
  • Fade animation: Uses CSS `transition` for smooth opacity changes when `.noshow` is added
  • Layout: Block layout with centered alignment; works across all browsers
